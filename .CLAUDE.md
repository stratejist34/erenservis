## 1. Brand Identity

**İsim:** Eren Servis — Şanzıman Uzmanı
**Konum:** Bostancı Oto Sanayi Sitesi, Kadıköy / İstanbul
**Sektör:** Otomatik şanzıman tamiri — DSG, ZF, CVT, S-Tronic, Tiptronic
**Konumlandırma:** Teknik uzmanlık. Piyasadaki genel servislerden farklı olarak,
  otomatik şanzıman mekatronik kart tamiri, basınç tüpü kontrolü ve
  komple revizyon gibi nadir bulunan yetkinliklere sahip niş uzman.

**Marka Vaadi (tek cümle):**
"Şanzımanınızı anlayan, teşhis koyan ve orijinal parçayla onaran uzman."

**Ton:** Güvenilir, teknik, açık sözlü. Satış baskısı yok, pazarlık dili yok.
  Bir cerrahın hasta yakınına konuştuğu netlik ve sakinlik.
  "Biz en iyiyiz" demez — "biz bunu yapıyoruz, işte sonuçları" gösterir.

**Hedef Kitle:**
- Birincil: 30-55 yaş, İstanbul Anadolu Yakası, araç sahibi erkekler.
  Otomatik şanzıman sorunu yaşıyor, Google'da "DSG arıza", "şanzıman titremesi"
  gibi semptom bazlı arama yapıyor. Teknik bilgiye değer verir ama uzman değil.
- İkincil: Sigorta ekspertizleri ve diğer servislerden yönlendirilen müşteriler.

## 2. Aesthetic Direction — "Industrial Precision"

**Konsept:** Endüstriyel atölye estetiği — karanlık, teknik, ham (raw) ama
kontrollü ve hassas. Bir CNC tezgahının arayüzü gibi: soğuk, kesin, güvenilir.
Lüks değil, gösterişli değil — ama ucuz veya amatör hiç değil.

**Referans atmosfer:** Karanlık bir oto servis atölyesinde, tezgahın üstündeki
tek lamba mekatronik kartı aydınlatıyor. Metal yüzeyler, yağ lekeleri yok ama
aletlerin düzeni cerrahi hassasiyette. Loş ama net.

### 2.1 Renk Sistemi

Mevcut sitedeki koyu lacivert + turuncu kombinasyonunu KORUYORuz ama
derinlik ve karakter ekliyoruz. Turuncu "varsayılan AI tercihi" gibi durmamalı —
endüstriyel uyarı rengi gibi durmalı (dikkat çekici, fonksiyonel).

```
/* Ana palet */
--bg-primary:      #0B0F19;    /* Derin gece mavisi, neredeyse siyah */
--bg-secondary:    #111827;    /* Koyu arduvaz — kart ve section arka planları */
--bg-elevated:     #1A2035;    /* Hafif yükseltilmiş yüzeyler */

/* Yüzey ve sınır */
--border-subtle:   #1E293B;    /* Kartlar arası ince çizgi */
--border-accent:   #334155;    /* Daha belirgin ayırıcılar */
--surface-glass:   rgba(255, 255, 255, 0.03); /* Cam efekti overlay */

/* Metin */
--text-primary:    #F1F5F9;    /* Ana metin — saf beyaz değil, göz yormaz */
--text-secondary:  #94A3B8;    /* İkincil açıklama metinleri */
--text-muted:      #64748B;    /* Üçüncül, caption seviyesi */

/* Aksan (CTA + vurgu) */
--accent-primary:  #F97316;    /* Endüstriyel turuncu — CTA, önemli vurgular */
--accent-hover:    #FB923C;    /* Hover state — hafif açılır */
--accent-glow:     rgba(249, 115, 22, 0.15); /* CTA arkasında yumuşak hale */
--accent-warm:     #FBBF24;    /* Amber — rakam vurguları, badge'ler */

/* Durum renkleri */
--success:         #22C55E;    /* Garanti, onay */
--info:            #3B82F6;    /* Bilgi notları */
```

**Renk kullanım kuralları:**
- `--accent-primary` (turuncu) YALNIZCA CTA butonları ve kritik vurgularda.
  Sayfada turuncu görüyorsan, tıklanabilir veya dikkat çekici olmalı.
  Dekoratif amaçla ASLA kullanılmaz.
- İkon renkleri turuncu DEĞİL — `--text-secondary` (gri) olmalı.
  Turuncu ikonlar aksanın gücünü seyreltir.
- Kart arka planları `--bg-secondary`, hover'da `--bg-elevated` geçişi.
- Beyaz metin (`--text-primary`) saf #FFFFFF değil, hafif mavi-gri.

### 2.2 Typography

```
/* Display / Başlık fontu */
font-family: 'Plus Jakarta Sans', system-ui, sans-serif;
/* Neden: Geometrik ama yumuşak, teknik hissi var ama soğuk değil.
   Inter'den daha karakterli, Space Grotesk'ten daha okunabilir.
   latin-ext subset Türkçe karakterleri destekler. */

/* Body / Gövde fontu */
font-family: 'Inter', system-ui, sans-serif;
/* Neden: Body text'te Inter kusursuz okunabilirlik sağlar.
   Display'de yasak olan Inter, body'de doğru seçim — amaç farklı.
   Karışıklık yok: başlık Jakarta Sans, gövde Inter. */
```

**Tipografik hiyerarşi:**
```
H1 (Hero):     48px / 700 weight / -0.02em tracking / 1.1 line-height
H2 (Section):  36px / 700 weight / -0.01em tracking / 1.2 line-height
H3 (Kart):     24px / 600 weight / normal tracking / 1.3 line-height
Body:          16px / 400 weight / normal tracking / 1.6 line-height
Caption/Small: 14px / 500 weight / 0.02em tracking / 1.4 line-height

Mobil ölçekleme: H1 → 32px, H2 → 28px, H3 → 20px
```

**Önemli:** Section üst etiketleri (ör. "UZMANLIK ALANLARIMIZ", "MÜŞTERİ YORUMLARI")
monospace veya sıkı tracking ile küçük, uppercase, `--accent-primary` renkte.
Bu endüstriyel etiket/damga hissi yaratır:
```
.section-tag {
  font-size: 12px;
  font-weight: 600;
  letter-spacing: 0.1em;
  text-transform: uppercase;
  color: var(--accent-primary);
}
```

### 2.3 Motion & Micro-interactions

**Felsefe:** Hareket = amaçlı, kısa, mühendislik hassasiyetinde.
Yavaş, kaypak, "rüyadayız" animasyonlar YASAK.
Her animasyon 200-400ms, easing: `cubic-bezier(0.16, 1, 0.3, 1)`.

- **Sayfa yüklenme:** Hero text staggered reveal (her satır 80ms delay).
  CTA butonları en son belirir (kullanıcı önce mesajı okusun).
- **Scroll reveal:** Intersection Observer ile, elementler hafif yukarı kayarak
  ve opacity ile belirir. Mesafe: 20px (fazla değil — abartılı parallax yok).
- **Kart hover:** `translateY(-2px)` + `border-color` geçişi (subtle, güven veren).
  Kartlar zıplamaz, dans etmez — hafifçe yükselir.
- **CTA hover:** Turuncu buton `--accent-hover`'a geçer + hafif glow efekti.
  `box-shadow: 0 0 20px var(--accent-glow)`.
- **Rakam sayaçları (15+, 5000+):** HAYIR. Animasyonlu sayaç "her template'de var"
  klişesi. Statik, güçlü, büyük rakam yeterli. Animasyon eklenmez.

**Performans kırmızı çizgisi:** CSS-only animasyonlar tercih edilir.
Framer Motion yalnızca staggered reveal gibi JS gerektiren durumlar için,
ve yalnızca `{ once: true }` ile — tekrar eden animasyon yok.

### 2.4 Backgrounds & Texture

Düz solid renkler YASAK. Derinlik olmalı ama performansı yememeli:

- **Hero arka plan:** CSS radial gradient + çok ince noise texture overlay.
  ```css
  background:
    radial-gradient(ellipse at 30% 50%, rgba(249, 115, 22, 0.06) 0%, transparent 60%),
    radial-gradient(ellipse at 70% 30%, rgba(59, 130, 246, 0.04) 0%, transparent 50%),
    var(--bg-primary);
  ```
  Hero'da fotoğraf kullanılacaksa: gerçek atölye/mekatronik kart fotoğrafı,
  üzerine koyu gradient overlay. Stok fotoğraf YASAK.

- **Section geçişleri:** `--bg-primary` → `--bg-secondary` arasında
  yumuşak geçiş. Sayfanın tamamı tek renk = monoton.

- **Kart yüzeyleri:** Hafif `backdrop-filter: blur()` ile glassmorphism
  DEĞİL — performans maliyeti yüksek. Bunun yerine:
  `background: var(--bg-secondary); border: 1px solid var(--border-subtle);`
  Hover'da border rengi `--border-accent`'e geçer.

- **İnce detay:** CSS `::before` pseudo-element ile grid pattern veya
  çok ince dot pattern arka plana eklenebilir (opacity: 0.03).
  Bu derinlik hissi yaratır ama dikkat dağıtmaz.

### 2.5 Layout & Spatial Composition

- **Asimetri kontrolle:** Hizmet kartları eşit 4'lü grid yerine,
  ilk 2 kart (DSG + ZF) daha büyük/prominent, son 2 (CVT + Bakım) daha küçük.
  Bu görsel hiyerarşi: "esas uzmanlığımız bunlar" mesajı verir.

- **Hero layout:** Merkeze hizalı metin çalışıyor ama can sıkıcı.
  Alternatif: sol tarafta metin + CTA, sağda mekatronik kart görseli
  veya stilize şanzıman illustrasyonu. Asimetrik ama dengeli.

- **Generous whitespace:** Section'lar arası minimum 120px padding.
  Sıkışık hissettirmeyen ama boş da durmayan dengeli alan.

- **Grid kırılımları:** Güvenin 6 Sebebi 3x2 grid yerine,
  ilk 3 büyük kart (üst sıra), alt 3 compact kart — veya
  horizontal scroll strip (mobilde zaten mantıklı).

## 3. Conversion Architecture — Eren Servis Specific

### 3.1 Primary CTA Strategy

**Birincil hedef:** Telefon + WhatsApp eşit ağırlıkta.

```
[📞 Hemen Arayın]     → tel:02165775151 (turuncu, filled, primary)
[💬 WhatsApp'tan Yazın] → wa.me/... (koyu, outlined, secondary)
```

**Yerleşim kuralları:**
- Hero: İki CTA yan yana (desktop), üst üste (mobil).
  Telefon CTA daima sol/üst (birincil pozisyon).
- Sticky header (scroll sonrası): Yalnızca telefon numarası + küçük WhatsApp ikonu.
- Sticky mobile footer bar: İki eşit buton — Ara + WhatsApp.
  Sayfanın her noktasında erişilebilir, ekranın alt 60px'ini kaplar.
- Her section sonunda: Sayfanın akışını kırmayan inline CTA.
  "DSG şanzımanınızda arıza mı var? Ücretsiz ön teşhis için arayın."

**Asla yapılmayacak:**
- Pop-up form. Otomotiv servis niche'inde conversion killer.
- "Fiyat teklifi al" formu hero'da. Kullanıcı fiyat öğrenmek istiyor ama
  form doldurmak istemiyor — telefon edip sormak istiyor.
- Üç veya daha fazla CTA bir arada. Karar felci yaratır.

### 3.2 Trust Signal Architecture

**Positioning:** Güven sinyalleri CTA'nın hemen altında veya yanında.
Kullanıcı CTA'yı gördüğü anda "bu adamlara güvenebilir miyim?" sorusu
aklına gelir — cevap 2 saniye içinde görünür olmalı.

**Hero altı trust bar (4 öğe, yatay strip):**
```
15+ Yıl Deneyim | 5000+ Tamamlanan İş | 2 Yıl Garanti | 7/24 Destek
```
Bu 4 öğe istatistik, güven, garanti ve erişilebilirlik sinyali.
Rakamlar animasyonlu sayaç DEĞİL — statik, bold, büyük.

**Google yorumları section'ı:**
- "5.0 Google Yorumları" badge'i prominent, CTA yakınında.
- Yorum kartlarında: ad, araç marka/model, onarım türü.
  "Mehmet K. — Audi A4 S-Tronic" formatı iyi, KORU.
- Yorum kartlarına fotoğraf eklenebilir (araç teslim anı, tamir detayı).

**Sertifika/yetkinlik göstergeleri:**
- Wolfstech DSG basınç tüpü — niş uzmanlık kanıtı.
- Marka logoları strip'i (VW, Audi, Seat, Skoda, Porsche, BMW, Nissan)
  — zaten var ama daha prominent olmalı, hero'ya yakın.

### 3.3 Page Flow — Conversion Funnel

Ana sayfa akışı, kullanıcının karar yolculuğunu takip eder:

```
1. HERO
   ↓ "Ne yapıyorlar?" — Ana mesaj + CTA + trust bar
2. HİZMETLER
   ↓ "Benim sorunumu çözebilirler mi?" — DSG/ZF/CVT kartları
3. NEDEN BİZ
   ↓ "Neden burayı seçmeliyim?" — 6 sebep (ama hiyerarşik)
4. SÜREÇ
   ↓ "Nasıl çalışıyor?" — 3-4 adım: Arama → Teşhis → Teklif → Onarım
   (Bu section mevcut sitede YOK — eklenmeli. Belirsizliği azaltır.)
5. YORUMLAR
   ↓ "Başkaları memnun mu?" — Google yorumları
6. BLOG / BİLGİ
   ↓ "Daha fazla bilgi" — Son 3 blog yazısı özeti
7. FOOTER + SON CTA
   ↓ "Tamam, iletişime geçeyim" — Harita + İletişim + son WhatsApp CTA
```

### 3.4 Eksik Section: "Nasıl Çalışır?"

Mevcut sitede bu yok ve büyük bir boşluk:

```
Adım 1: Bizi Arayın
  "Şanzıman sorununuzu anlatın, ön bilgi verelim."

Adım 2: Ücretsiz Ön Teşhis
  "Aracınızı teslim alın, bilgisayarlı diagnostik yapılsın."

Adım 3: Net Fiyat Teklifi
  "Teşhis sonrası yazılı, sürprizsiz fiyat teklifi."

Adım 4: Onarım ve Teslim
  "Orijinal parça ile onarım, garanti belgesi ile teslim."
```

Bu akış kullanıcının "ne olacak bilmiyorum" korkusunu ortadan kaldırır.
Gestalt prensiplerinden Closure — süreç tamamlanmış hissettirmeli.

## 4. Component-Specific Guidelines

### Hero
- Sol hizalı veya merkezi metin — ikisi de olabilir, asimetri tercih edilir.
- "Bostancı'da Otomatik Şanzıman Tamiri ve Servisi" → bu H1 iyi, KORU.
  SEO ve kullanıcı açısından doğru.
- "Otomatik Şanzıman" kelimesi turuncu — vurgu doğru, KORU.
- Alt metin: "DSG, ZF, CVT ve tüm otomatik şanzıman türlerinde 15 yılı aşkın deneyim."
  → İyi ama "15 yılı aşkın deneyim" bold olmalı (Von Restorff).
- Badge'ler (DSG ve CVT Uzmanı, Orijinal Parça, Garantili Hizmet):
  Koyu chip/tag formatı iyi — ama turuncu ikon yerine `--text-secondary` ikon.

### Hizmet Kartları
- 4 eşit kart yerine: DSG kartı daha büyük (span-2 veya daha yüksek),
  ZF kartı orta, CVT ve Bakım kartları küçük.
  Uzmanlık hiyerarşisi görsel olarak yansımalı.
- Her kartta: tag'lar (DQ200, ZF 8HP, Nissan vb.) iyi — KORU.
- "Detaylar →" link'i yerine daha spesifik: "DSG Tamir Detayları →"

### Müşteri Yorumları
- Horizontal scroll çalışıyor ama auto-play carousel YASAK.
  Kullanıcı kontrolü olmalı (drag veya ok butonları).
- Her kartta araç bilgisi (Audi A4 S-Tronic, Skoda Octavia DSG) — harika,
  bu sektörel güven sinyali. KORU ve öne çıkar.

### Footer
- WhatsApp CTA footer'da var — iyi.
- Harita embed değil, statik harita görseli + "Google Maps'te Aç" linki.
  (iframe embed = performans maliyeti + CLS riski)
- Çalışma saatleri prominent: Pzt-Cmt 08:00-18:00.

## 5. Anti-Patterns — Bu Projede YAPMA

- Turuncu rengi dekoratif ikonlarda kullanma. CTA gücünü seyreltir.
- Animasyonlu sayaç/counter kullanma. Her template'de var, klişe.
- Stok fotoğraf kullanma. Gerçek atölye, gerçek teknisyen, gerçek iş.
  Stok yoksa hiç fotoğraf olmasın — CSS ile atmosfer yarat.
- Pop-up, modal, veya girişte form gösterme. Bu sektörde güven kırar.
- "En iyi", "1 numara", "lider" gibi doğrulanamayan iddialar. Yasal risk.
  Bunun yerine: "15 yıllık deneyim", "5000+ tamamlanan iş" — ölçülebilir.
- Sahte urgency (sayaç, "son 3 randevu kaldı" vb.). Güven kırar.
- Tüm kartların aynı boyutta olması. Hiyerarşi yok = her şey eşit önemde
  görünür = hiçbir şey önemli görünmez.
- Google Fonts harici yükleme. `next/font` ile self-host.
- Map iframe embed. Statik görsel + link kullan.

## 6. Lighthouse & Performance Targets

Bu proje için spesifik hedefler (Türkiye 4G mobil baz):

| Sayfa | LCP Hedef | JS Budget (ek) | CLS |
|-------|-----------|-----------------|-----|
| Ana Sayfa | < 1.8s | < 40KB | < 0.05 |
| Hizmet Detay | < 1.5s | < 30KB | < 0.05 |
| Blog Yazı | < 1.2s | < 20KB | < 0.02 |
| İletişim | < 1.5s | < 35KB | < 0.05 |

Ana sayfanın JS budget'ı en yüksek çünkü yorum carousel ve
sticky mobile CTA bar gibi interaktif elementler var.
Blog sayfaları neredeyse sıfır JS olmalı — saf statik.
